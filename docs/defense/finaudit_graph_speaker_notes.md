# FinAudit-Graph 答辩 PPT 讲稿

建议总时长：6-10 分钟。每页只讲一个结论，演示页按现场系统结果展开。

## 1. 封面

开场直接点明：这个项目不是单次大模型调用，而是围绕财务审计场景，把数据、模型、应用和自动化复核串起来的全流程项目。

## 2. 项目背景与目标

这一页按评分标准说：项目解决财务审计初筛、风险识别和复核流转问题；同时说明为什么它能作为全流程项目展示。

## 3. 全流程总览

按“数据标注 → 模型训练 → 应用开发 → Neo4j图谱 → N8N自动化”的顺序讲，强调每一环都有文件证据或可运行演示。

## 4. 技术架构图

这一页重点讲清楚调用关系：Streamlit 面向演示，FastAPI 做服务入口，LangGraph 编排审计节点，Chroma RAG 补审计准则，Neo4j 负责关联方关系写入和穿透查询，Agent 支撑风险分析，N8N 负责高风险复核通知。

## 5. 数据标注与模型训练

展示 Label Studio 截图和 LoRA 训练曲线。说明当前 LoRA 是训练成果和可选增强，不夸大成主流程唯一决策源。

## 6. 核心功能演示 1

现场演示时先打开 Streamlit，再上传 realistic_business_financials.xlsx，强调测试数据已经从小样例升级成更贴近业务的数据。展示 Neo4j/图谱状态时要说明：配置 Neo4j 会写入并查询图数据库，现场不可用时自动回退 JSON，保证演示稳定。

## 7. 核心功能演示 2

展示报告生成，然后切到 N8N 工作流和邮箱结果。重点说清楚 webhook payload、条件判断和邮件通知闭环。

## 8. 技术亮点与创新点

这一页不要逐个念技术栈，而要讲“为什么这样组合”：Chroma RAG 补审计准则，Neo4j 图谱补关联方关系，LangGraph 控流程，N8N 把审计结果变成业务动作。

## 9. 个人贡献与收获

按五个环节讲个人贡献，最后落到工程能力：数据、模型、应用、自动化、测试都能自己串起来。

## 10. 演示路线与答辩收束

如果没有视频，就按这页做现场演示清单。答辩结束时强调：系统可运行、Neo4j 图谱有真实写入/检索能力且有 fallback，证据链完整、加分项明确。
