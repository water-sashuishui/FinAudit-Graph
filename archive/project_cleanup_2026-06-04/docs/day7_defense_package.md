## 常见答辩问题

| 问题 | 建议回答 |
| --- | --- |
| 为什么使用 LangGraph？ | 审计流程天然是多步骤状态流转，LangGraph 能明确表达节点、状态和流转顺序，便于解释和调试。 |
| 微调是否真的必要？ | 微调用于增强审计风险分类和专业表达；冲刺原型用小样本证明流程，生产版本需要更大规模真实审计语料。 |
| Neo4j 的作用是什么？ | 图数据库适合表达股权、控制权、供应商和资金往来关系，能支持关联方穿透查询。 |
| RAG 解决什么问题？ | RAG 将审计准则和法规依据带入模型判断。当前已实现本地持久化向量库，检索结果带有相似度和 `retrieval_mode: vector`，提升可解释性并减少纯模型幻觉。 |
| 当前系统的局限是什么？ | 样本数据规模有限，真实 PDF 解析、鉴权、模型评估和飞书联动仍需生产化完善。 |
